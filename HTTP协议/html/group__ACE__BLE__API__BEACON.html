◆ acebt_start_beacon_scan()

Interface to request beacon manager to initiate a BLE scan on behalf of a client. If API returns BT_SUCCESS, a new scaninstance will be created and filled on out parameter scanInstance. This scan instance should be used to control this scan in all subsequent calls. Whenever status of the scan request changes scanStateChanged() callback of aceBT_beacon_callbacks_t will invoked.

Parameters
sessionHandleThis handle should be same as handle returned by OpenSession() API
clientIdclient identification assigned for this client assigned by Beacon manager If no ID is currently assigned, pass 0xFFFF.
settingsaceBT_BeaconScanSettings_t filled with the application requested scan settings for this scan request
filtersaceBT_BeaconScanFilterList_t filled with the application requested scan filters for this scan request
scanInstanceOut parameter that will be filled with reference to this scanning instance.
Returns
BT_SUCCESS if success; otherwise, appropriate aceBT_status_t error.

◆ aceBT_startBeaconScan()

aceBT_status_t aceBT_startBeaconScan ( aceBT_sessionHandle  sessionHandle,
aceBT_BeaconClientId  clientId,
aceBT_BeaconScanSettings_t  settings,
aceBT_BeaconScanFilterList_t *  filter_lists,
uint8_t  num_filter_lists,
aceBT_scanInstanceHandle *  scanInstance 
)

Interface to request beacon manager to initiate a BLE scan on behalf of a client. If API returns BT_SUCCESS, a new scaninstance will be created and filled on out parameter scanInstance. This scan instance should be used to control this scan in all subsequent calls. Whenever status of the scan request changes scanStateChanged() callback of aceBT_beacon_callbacks_t will invoked.

Parameters
[in]sessionHandleSession handle for the current Bluetooth client
[in]clientIdclient identification assigned for this client assigned by Beacon manager If no ID is currently assigned, pass 0xFFFF.
[in]settingsaceBT_BeaconScanSettings_t filled with the application requested scan settings for this scan request
[in]filter_listsaceBT_BeaconScanFilterList_t filled with the application requested scan filters for this scan request
[in]num_filter_listsNumber of filter lists in previous parameter's pointer
[out]scanInstanceOut parameter that will be filled with reference to this scanning instance.
Returns
ACEBT_STATUS_SUCCESS if success
ACEBT_STATUS_NOMEM if ran out of memory
ACEBT_STATUS_BUSY if profile is busy connecting another device
ACEBT_STATUS_PARM_INVALID if request contains invalid parameters
ACEBT_STATUS_NOT_READY if server is not ready
ACEBT_STATUS_UNSUPPORTED if does not support BLE
ACEBT_STATUS_FAIL for all other errors
