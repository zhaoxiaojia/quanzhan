◆ aceLogHal_id_t

enum aceLogHal_id_t

Log buffer ID. It's up to the platform on how to map buffer ID to its implementation. They can be mapped to one buffer or file, or individual buffers or files. On platforms that have sufficient resources, the implementation can map each log buffer ID to an individual log buffer or file. On platforms that have limited resources, the implementation can map multiple buffer IDs to one or a few buffers or files.

Enumerator
ACE_LOGHAL_ID_MAIN 

For apps and middlewares.

ACE_LOGHAL_ID_SYSTEM 

For middleware, dpk and platform modules.

ACE_LOGHAL_ID_CRASH 

For crash log from the platform or run time systems.

ACE_LOGHAL_ID_DM_MAIN 

For device manufacture modules and apps.

ACE_LOGHAL_ID_BIN 

For binary logs. The support for the binary log is optional.

ACE_LOGHAL_ID_REMOTE 

Reserved for remote devices.

ACE_LOGHAL_ID_MAX 

Number of the logs.

◆ aceLogHal_level_t

enum aceLogHal_level_t

Log severity levels.

Enumerator
ACE_LOGHAL_VERBOSE 

Even more detailed informational messages than the ACE_LOGHAL_DEBUG.

ACE_LOGHAL_DEBUG 

Detailed informational messages that's useful for debugging an application.

ACE_LOGHAL_INFO 

Informational messages that highlight the progress of the application.

ACE_LOGHAL_WARN 

Recoverable conditions that put the application in an undesirable state.

ACE_LOGHAL_ERROR 

Unrecoverable error conditions that still allow the application to continue.

ACE_LOGHAL_FATAL 

Severe error condition that will likely lead the application to abort.

◆ aceLogHal_platformProperty_t

enum aceLogHal_platformProperty_t

Properties of the platform implementation. These are used as keys to the aceLogHal_getPlatformProperty() API.

Enumerator
ACE_LOGHAL_PROPERTY_API_VER 

API version. Implementation should return ACE_LOGHAL_API_VERSION

ACE_LOGHAL_PROPERTY_LOG_BIN_MSG_MAX_LEN 

Max size of the binary log message. Returns 0 for no limit.

ACE_LOGHAL_PROPERTY_LOG_TXT_MSG_MAX_LEN 

Max size of the log message. Log messages larger than this may be truncated. This value will be used by the ace_log module to allocate memory for the log from the caller's stack.

ACE_LOGHAL_PROPERTY_LOG_TXT_ENTRY_MAX_LEN 

Max size of the formatted log entry. This value can be used by the application to allocate memory to read log entries.

ACE_LOGHAL_PROPERTY_SUPPORTED_READ_MODES 

Bitmask of the supported aceLogHal_readMode_t.

◆ aceLogHal_readMode_t

enum aceLogHal_readMode_t

Modes for aceLogHal_read().

Enumerator
ACE_LOGHAL_READ_SINGLE 

Read one log message or entry. Support for this is required.

ACE_LOGHAL_READ_FILL 

Fill the read buffer with as many complete log entries as the buffer will allow. Support for this is optional.
