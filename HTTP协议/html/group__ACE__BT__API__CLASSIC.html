[out]p_uuidsArray of aceBT_uuid_t
[out]p_uuid_countLength of aceBT_uuid_t array
Returns
ACEBT_STATUS_SUCCESS if success
ACEBT_STATUS_NOMEM if ran out of memory
ACEBT_STATUS_BUSY if profile is busy connecting another device
ACEBT_STATUS_PARM_INVALID if request contains invalid parameters
ACEBT_STATUS_NOT_READY if server is not ready
ACEBT_STATUS_UNSUPPORTED if does not support classic BT
ACEBT_STATUS_MAX_BUFF if reached buf_size
ACEBT_STATUS_FAIL for all other errors

◆ aceBT_startDeviceDiscovery()

aceBT_status_t aceBT_startDeviceDiscovery ( aceBT_sessionHandle  sessionHandle)

Function to asynchronously start a Bluetooth device discovery procedure. The state of the discovery will be known by the clients via /ref adapter_discovery_state_cb(). The devices found via discovery will be sent as individual callbacks of /ref device_discovered_callback() passed as param.

Clients should note that the /ref device_discovered_callback() will be invoked multiple times as the local Bluetooth device discovers each device. Also, the callback can also be triggered after stop discovery is called since RNR (HCI remote name request) can take longer sometimes.

Parameters
[in]sessionHandlesession handle for the current Bluetooth session
Returns
ACEBT_STATUS_SUCCESS if success
ACEBT_STATUS_NOMEM if ran out of memory
ACEBT_STATUS_BUSY if profile is busy connecting another device
ACEBT_STATUS_PARM_INVALID if request contains invalid parameters
ACEBT_STATUS_UNSUPPORTED if does not support classic BT
ACEBT_STATUS_FAIL for all other errors

API implementation to start device discovery. This is an asynchronous call and the device found callbacks will be delivered to all clients that invoke this API within a discovery window.
