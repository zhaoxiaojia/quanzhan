Post a message to a dispatcher with brief blocking.

This function posts a message to the queue of a dispatcher blocking briefly for the operation to succeed. Upon dequeue, the module's on_msg callback is called.

Warning
Care should be taken while allocating memory to which msg points. If the caller is blocking on the response from dispatcher, the msg may be placed on the stack, else if the post triggers asynchronous actions, its best to allocate memory from the heap or mempool.
Do not block for results of execution when posting from within a dispatcher context.
Parameters
[in]hModule dispatch handle
[in]eventEvent to post to the dispatcher
[in]msgPointer to the message
[in]lenLength of the message
Returns
ACE_STATUS_OK on success, else error code from ace_status_t.

aceDispatcher_work_t *  w 
)

Post work to a dispatcher.

This function posts work to the queue of a dispatcher for execution in FIFO order. If the user wishes to be able to cancel the work in the future, the same work pointer serves as a handle to refer to the deferred work. For such use-cases avoid stack allocation for the work struct. On the other hand, if the user's intent is to schedule work execution without delay and the need to cancel in the future, the work struct can be on the user's stack.

Parameters
[in]hPointer to module dispatch handle
[in]wPointer to the work to be dispatched
Returns
ACE_STATUS_OK on success, else error code from ace_status_t.
