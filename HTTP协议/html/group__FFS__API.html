ace_status_t aceFfs_deinit ( void  )

De-initialize the FFS subsystem. If the provisionable is started, it will do a stop first. This API is re-entrant.

Returns
ACE_STATUS_OK if everything goes well.
ACE_STATUS_UNINITIALIZED if not initialized.

◆ aceFfs_init()

ace_status_t aceFfs_init ( aceFfs_config_t *  config)

Initialize the FFS provisionable subsystem.

Parameters
[in]config,configurationparameters. This is optional. If no configuration is supplied, default advertisement and provisioning timeout will be used which is ACE_FFS_DEFAULT_ADVERTISING_TIMEOUT seconds and ACE_FFS_DEFAULT_PROVISIONING_TIMEOUT seconds respectively.
Returns
ACE_STATUS_OK if everything goes well.
ACE_STATUS_GENERAL_ERROR for other errors.

◆ aceFfs_start()

ace_status_t aceFfs_start ( aceFfs_mode_t  mode)

Start FFS Provisioning. This starts the BLE device beaconing and awaits for provisioner connection. This API is re-entrant.

Parameters
modeFFS provision modes (Add the individual modes together to form combinations. i.e., ACE_FFS_MODE_ZTS | ACE_FFS_MODE_UGS)
Returns
ACE_STATUS_OK if everything goes well.
ACE_STATUS_UNINITIALIZED if not initialized.
ACE_STATUS_BAD_PARAM if invalid mode is provided.
ACE_STATUS_GENERAL_ERROR if start failed.

◆ aceFfs_stop()

ace_status_t aceFfs_stop ( void  )

Stop FFS Provisioning. This will do a force stop of the provisionable workflow regardless of which stage it is in. This API is re-entrant.

Returns
ACE_STATUS_OK if everything goes well.
ACE_STATUS_UNINITIALIZED if not initialized.
ACE_STATUS_GENERAL_ERROR if stop failed.
