aceBT_status_t aceBT_openSession ( aceBT_sessionType_t  session_type,
aceBT_sessionCallbacks_t *  callbacks,

Opens/Creates a Bluetooth session for an Application Main entry point for Bluetooth APIs. Clients will need to invoke this API to create a Bluetooth session. Once the session is successfuly opened,client application should register into Bluetooth classic or LE modules based on the session availablity. Use aceBT_getSupportedSession API to retrieve supported Bluetooth session type by underlying platform.

Parameters
[in]sessionTypeType of session (either dual mode or LE only) to be opened
[in]callbackspointer to session callbacks
[out]sessionHandlewill be updated with session handle if session creation was successful.
Returns
ACEBT_STATUS_SUCCESS if success
ACEBT_STATUS_NOMEM if ran out of memory
ACEBT_STATUS_BUSY if profile is busy connecting another device
ACEBT_STATUS_PARM_INVALID if request contains invalid parameters
ACEBT_STATUS_NOT_READY if server is not ready
ACEBT_STATUS_FAIL for all other errors

Main entry point for Bluetooth APIs. Clients will need to invoke this API to create a Bluetooth session. Use aceBT_getSupportedSession API to retrieve supported Bluetooth session type by underlying platform
