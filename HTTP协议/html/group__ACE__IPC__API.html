◆ aceIpc_getCharArrayProperty()

aceIpc_status aceIpc_getCharArrayProperty ( aceIpc_handle  handle,
const char *  source,
const char *  propertyName,
char **  charArray,
int *  charArraySize 
)

Gets charArray property from a given publisher for given property name. Returns an error if there is an error. An error could be that the property is not published by the publisher, the publisher is not registered on the bus at the moment or the property does not support the requested access method failed.

It could also return an error sent by the remote process. These errors start at: aceIpc_propErrStart

Parameters
handle[IN] aceIpc handle
publisher[IN] Unique name of the remote process that publishes the property
propertyName[IN] Name of the property that needs to be accessed
charArray[OUT] The returned charArray by remote process. This is only valid if there is no error set. If this charArray is valid, it must be freed by calling aceIpc_freeCharArray as the array is allocated by aceIpc.
charArraySize[OUT] The size of returned charArray by remote process. This is only valid if there is no error set. The caller must pass in a pointer to a valid integer during the call, it's not allocated by aceIpc.
Returns
Returns aceIpc_errNone on success

◆ aceIpc_open()

aceIpc_handle aceIpc_open ( const char *  serviceName)

Opens a handle. This must be first call before using the API further.

serviceName is used as the "source" when sending events. It is also used as the "publisher" when publishing properties

serviceName cannot be longer than ACEIPC_NAMES_MAX_LEN, and must be unique in the system. If there is another entity using the same service name, this call will return an error.

Note: Some additional restrictions that apply to serviceName

  • serviceName must only contain the ASCII characters "[A-Z][a-z][0-9]" with "-" discouraged.
  • serviceName must contain at least one '.' (period) character (and thus at least two elements).
  • serviceName must not begin with a '.' (period) character.
  • elements in serviceName must not begin with a digit.
  • The recommanded name would be something like "com.lab126.service".
Parameters
serviceName[IN] service name
Returns
A non-NULL handle on success. NULL on failure.

◆ aceIpc_unsubscribeExt()

aceIpc_status aceIpc_unsubscribeExt ( aceIpc_handle  handle,
const char *  publisher,
const char *  eventName,
void **  userData 
)

Unsubscribes to a specific event or all events from a given source.

Parameters
handle[IN] aceIpc_handle obtained by calling aceIpc_open
publisher[IN] Publisher of the events to unsubscribe from. This parameter cannot be NULL
eventName[IN] Event name to unsubscribe from. If this parameter is NULL it will unsibscribe from a matching subscription to all events, but will not unsubscribe to other specific events in the same source.
userData[OUT] The userData parameter provided to 'subscribe' above will be returned in this location, if it is not null.
Returns
aceIpc_errNone if the subscription is successful

Unsubscribe from a publisher/event pair. This removes the DBus match rule, and optionally returns the 'userData' pointer provided during subscription.

Parameters
handlethe aceIpc_handle returned when aceipc was opened
publisherthe publisher id to unsubscribe from
eventNamethe event name to unsubscribe from. NULL to remove a subscription to all events. NOTE: this means that the match rule for 'all' is removed, not that all match rules for the pusher are removed.
userDataIf not null, the userData parameter passed in LipcSubscribeEx is returned here.
