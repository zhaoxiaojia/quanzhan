file  aceAIPC_api.h
 AIPC is a new RPC/IPC system designed for ACE to meet both security and performance requirement AIPC provides super high speed RPC & IPC by using specially designed shared memory AIPC provides synchronized / asynchronized RPC to support ACE middleware / app to call function on remote process just as to call local function. AIPC provide callback feature support in RPC, it support ACE middleware and ACE application to use interprocess process callback mechanism just as locally. AIPC provides event dispatcher mechanism to support ACE middleware and ACE application to subscribe or unsubscribe to events and publish or receive events automatically AIPC provides client and server authorization / authentication support to enforce security AIPC provides a full set of easy to use interface with long term support. AIPC API is complete and extendible to meet more advanced features in the future without changing API's signature AIPC automatically handle RPC/IPC marshalling and demarshalling, routing, enqueeuing, dequeening, event dispatching, subscriber and publisher management and etc. AIPC automatically create worker thread and handle the process/thread synchronization across the whole ACE environment AIPC automatically manage the life status of clients and servers to maintain the reliable and stable system wide RPC and IPC. AIPC automatically manage the resource for client and servers and handle resource initialization, allocation, deallocation and full cleanup AIPC automatically manage the security auditing of the RPC & IPC AIPC automatically select the best IPC physical channel: such as shared memory, domain socket or serial bus based on the client server role, performance and security requirement.
