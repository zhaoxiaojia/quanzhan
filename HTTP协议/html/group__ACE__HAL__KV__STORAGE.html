Detailed Description

HAL API for Key value storage.

The Key Value storage APIs provide a way for the middlewares to be able to store configurations, information on persistent storage. Unlike files there is no access to sub parts of value making this suitable only for small pieces of data, configuration parameters.

KVS keys have the format of X.Y where X is the group name and Y is the key name. If X.Y key format is provided to KVS HAL APIs, it assumes that user would like to access the key Y in group X. If no group name is specified, the assumption is that the key Y is to be accessed from default group (pre-defined). Note that, the entire string after the first '.' should be treated as the key name. For example "somegroup.somekey_subkey", "somegroup" is the group name and "somekey_subkey" should be treated as the key name. No "."(dot) is allowed in group key and key name.

Note
The length limit of KVS group name is ACE_KV_STORAGE_HAL_MAX_GROUP_LEN.
The length limit of KVS key name is ACE_KV_STORAGE_HAL_MAX_KEY_LEN.

◆ aceKeyValueDsHal_listKeys()

ace_status_t aceKeyValueDsHal_listKeys ( const char *  group_name,
aceKeyValueDsHal_listKeys_cb  list_keys_cb 
)

List key names within a KVS group.

Caller needs to provide a callback (aceKeyValueDsHal_listKeys_cb type). This is a synchronous call. The callback will NOT be triggered after the function returns.

Example:

// Define the callback function
static int listKeys_cb(const char* key_name) {
printf("%s\n", key_name);
return 0;
}
// Call aceKeyValueDsHal_listKeys function with callback specified
ace_status_t rc = aceKeyValueDsHal_listKeys(group_name, &listKeys_cb);
Parameters
group_name[in]Pointer to string for the group name.
list_keys_cb[in]Callback used when aceKeyValueDsHal_listKeys API gets each key name.
Returns
: ace_status_t: Return ACE_STATUS_OK if successful, or any non-zero ace_status_t error code.
