◆ aceFfs_provisionerInit()

ace_status_t aceFfs_provisionerInit ( const aceFfs_provisionerConfig_t *  config)

Initialize the FFS provisioner subsystem. BLE provisioner will try to initialize Wifi client and wait for maximum of 10 seconds before giving up. To avoid the delay, one can ensure the initting of BLE provisioner is called after the system Wifi service is up and ready. [in] config configuration parameters. The config and callbacks in it are optional. While optional it's recommend to be provided so that the user gets notified about any state change.

Returns
ACE_STATUS_OK if everything goes well.
ACE_STATUS_GENERAL_ERROR for other errors.

ace_status_t aceFfs_provisionerStop ( aceFfs_provisionerMode_t  mode)

Stop FFS Provisioner. This will do a force stop of the provisioner workflow regardless of which stage it is in. This API is reentrant. [in] mode specify FFS provisioner mode (over BLE or WiFi).

Returns
