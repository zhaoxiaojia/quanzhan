Detailed Description

Button HAL API Definitions.

Button HAL defines a set of button abstraction APIs to handle the hardware button activities such as button pressed event, hardware button configurations. These button HAL APIs are used by Button Manager Middleware which polling the button event and handle the button event accordingly.

Steps to setup and use Button Hal interface: 1) aceButtonHal_open() needs to be called first to have button Hal APIs ready for use. 2) Use aceButtonHal_getNumButtons() to query the total number of buttons available on platform. 3) Use number of button info from previous step to allocate enough memory for retrieving aceButtonHal_info such as button index and name via aceButtonHal_getInfo(). 4) Activate (enable) the button and register the callback via aceButtonHal_setParams(). By default, all buttons are disabled at beginning. aceButtonHal_getParams() can be use to retrieve current activation state.
