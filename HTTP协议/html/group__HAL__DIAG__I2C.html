◆ aceDiagI2cHal_callback_t

typedef void(* aceDiagI2cHal_callback_t) (aceDiagI2cHal_opStatus_t op_status, void *user_context)

The callback function for completion of I2C operation. This callback funcion is passed to diag I2C hardware abstraction layer adapter by using aceDiagI2cHal_readAsync() or aceDiagI2cHal_writeAsync(). It shouldn't be invoked in ISR directly.

Parameters
[out]op_statusI2C asynchronous operation status.
[in]user_contextUser Context passed when setting the callback. This is not used or modified by the driver. The context is provided by the caller when setting the callback, and is passed back to the caller in the callback.

◆ aceDiagI2cHal_readAsync()

aceDiagI2cHal_callback_t  callback,
const void *  user_context,
size_t  bytes,
uint8_t *  buffer 
)

Reads the requested bytes from the data address (register address) of the specified I2C slave device through I2C bus in non-blocking mode.

Parameters
[in]i2c_descriptorThe pointer to aceDiagI2cHal_desc_t instance returned in aceDiagI2cHal_open() call.
[in]slave_addrThe I2C slave address.
[in]data_addrThe I2C data address (register address).
[in]callbackThe callback function to be invoked after read completed.
[in]user_contextThe user context to be passed back when callback is called. user_context should be allocated by caller. The life cycle of user_context should be maintained until callback function is invoked by diag hardware abstraction layer adapter.
[in]bytesThe number of bytes to read.
[out]bufferThe receive buffer for the data read. Buffer should be allocated by caller. The life cycle of buffer should be maintained until callback function is invoked by diag hardware abstraction layer adapter.
Return values
ACE_STATUS_OKBus not busy and succesfully start read command.
ACE_STATUS_BUSYBus busy.
ACE_STATUS_DEVICE_OPERATION_ERRORLow level driver error.
