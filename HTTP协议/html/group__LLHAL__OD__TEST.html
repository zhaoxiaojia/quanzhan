Introduction

This document is to provide an overview and instructions on porting and running the on-device tests for IOT LL-HAL. The on-device testing relies on Unity test framework to do the verification and to invoke the tests through the CLI. These tests run on target device to test LL-HAL functionality and performance.

Test Components

  • Unity - A test framework written in C especially for embedded software. It's mainly used for test assertion and its test macros.
  • Test Runner/Test Main - Test runner is where tests get called and where we setup board config for the board under test.
  • Test CLI - This is how to invoke tests. It is either called to run all tests, component-specific tests, or component-specific "assisted" tests. This document does not cover the latter tests, which needs interaction with some external board to for interacting with the DUT's peripherals.
  • Test Cases - Every HAL component has its own tests and within each test there will be test cases for that component.
  • Test Board Config - Since LL-HAL is a generic interfaces, we need to provide a way to configure the test to fit different IOT platforms/boards. Board config is where we set different hardware configurations for every device. It also enables setting the configuration for the different use cases.

Tests Runner/Test Main
