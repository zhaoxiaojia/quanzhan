Detailed Description

OS Abstraction Layer.

Warning
On the Android platform, the implementation of Shmem and IPMutex in OSAL depends on the shmd daemon. The application using Shmem or IPMutex needs to apply specific sepolicy to communicate with the shmd daemon. OSAL will initialize Shmem and IPMutex components during linking, and it also needs to communicate with the shmd daemon. The application which is using other functionalities, except Shmem and IPMutex, will also need the specific sepolicy by default in order to link OSAL.

We have designed a mechanism to disable the Shmem and IPMutex initialization when linking OSAL if the application is not using Shmem and IPMutex.

We use an environment variable (SHMLIB_DISABLE_ONLOAD) to identify whether the application linking OSAL is using Shmem or IPMutex. User can pass this environment variable to the application when starting it in order to disable the Shmem and IPMutex initialization. Thus, there will be no sepolicy violation(s) occurring during linking OSAL if the application does not apply the specific sepolicy.

To apply sepolicy, refer to device/amazon/common/sepolicy/common/ace/te_macros.

To disable Shmem and IPMutex initialization, refer to ace/middleware/metricsd/ace_metricd.rc.
