Detailed Description

This file contains all needed symbols for consumers of the cross platform events API's in ACE.
Refer ace_global_events_list.h for list of events.
Its goal is to support multicast(One to Many) events-
In Linux - a wrapper around ACE IPC
In RTOS - a library which is run in the publishers context.
.

◆ aceEvents_init()

ace_status_t aceEvents_init ( const char *  EvtModuleName)

Initialize the events module.

Note
This API must be called before any other Events API. Not thread-safe.
EvtModuleName cannot be longer than ACE_EVENTS_NAMES_MAX_LEN and must be unique in the system.If there is another entity using the same service name, error will be returned. A recommended name would be something like "com.xxx.xxxx".
Parameters
[in]EvtModuleName(Linux-only)is the name of a publisher/subscriber.
Returns
ACE_STATUS_OK on success, else one of the error codes in ace_status_t
Note
aceEvents_init should be called only once at the init stage of a process.
