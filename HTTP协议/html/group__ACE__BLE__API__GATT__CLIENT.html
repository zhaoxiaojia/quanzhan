◆ aceBT_bleBeginReliableWriteCharacteristics()

aceBT_status_t aceBT_bleBeginReliableWriteCharacteristics ( aceBT_bleConnHandle  connHandle)

API Initiates a reliable write transaction for a given remote device.

All reliable write operation performed using this API will be acknowledged with on_ble_write_characteristics_callback, the server will echo the data that is originally written so that it can be validate to continue the operation. Once the write operation is completed the the transaction is committed by invoking #aceBT_bleExecuteReliableWrite or aborted using #aceBT_bleAbortReliableWrite

Parameters
[in]connHandleconnection handle
Returns
ACEBT_STATUS_SUCCESS if success
ACEBT_STATUS_NOMEM if ran out of memory
ACEBT_STATUS_BUSY if profile is busy connecting another device
ACEBT_STATUS_PARM_INVALID if request contains invalid parameters
ACEBT_STATUS_NOT_READY if server is not ready
ACEBT_STATUS_FAIL for all other errors
