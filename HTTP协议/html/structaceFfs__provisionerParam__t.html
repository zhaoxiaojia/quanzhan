Detailed Description

Per mode configuration to initialize FFS provisioner Those callbacks may be called in another thread env, protection needs to be done by the implementation of these callbacks.

◆ provisioner_get_configuration

int(* aceFfs_provisionerParam_t::provisioner_get_configuration) (aceFfs_provisionerMode_t mode, aceFfs_provisionerConfigParam_t config, char *buffer, size_t buffer_size)

Called to get device configuration. Fills in the buffer with the string of the requested configuration value. The caller should allocate the required memory with the configuration max length definition. Callee must not write more than the passed maximum buffer size. (ACE_FFS_PROVISIONER_DEVICE_CONFIGURATION_MAX_LEN).

Parameters
[in]modeFFS provisioner mode
[in]configenum type of configuration requested
[in,out]bufferpointer to the output buffer (caller allocated)
[in]buffer_sizebuffer size of the content output buffer
Returns
Returns write length on success, and a negative value otherwise. Failures shall result in default configuration value being used.
