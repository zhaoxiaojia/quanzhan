file  hal_ambientLightSensor.h
 ACE ALS (Ambient Light Sensor) HAL builds on top of platform specific ALS device driver. It enables the upper layer software to retrieve sensor lux value from ambient light sensor.

file  hal_audio_mgr.h
 
file  hal_battery.h
 ACE Battery HAL provides the interfaces required for low level code to update battery properties and health to middleware. This layer will also serve the commands/requests from the Battery MW Service.
 
file  hal_block_device.h
 Block Device HAL provides an interface for access to block-based storage.
 
file  hal_button_mod.h
 ACE Button HAL builds on top of the device specific driver. It keeps track of activation state and handles UP/DOWN changes. Any changes are passed on to the button manager. The HAL is not responsible for timing or more complex state machine-like buttons - that's the responsibility of the manager.
 
file  hal_circularDs.h
 ACE Circular Data Store HAL implements circular buffer based storage in flash for its users.
 
file  hal_device_info.h
 Device Info HAL provides access to Device Info data.
 
file  hal_dha.h
 The DHA HAL builds on top of platform specific DHA/Security implementation and provides security and registration function access for middleware. Supports DHAv2 only.
 
file  hal_fr.h
 ACE Factory Reset HAL builds on top of platform specific factory reset callbacks.
 
file  hal_kv_storage.h
 Key-Value Storage (KVS) HAL provides an interface to persistently store and retrieve data on the target platform.
 
file  hal_led.h
 ACE LED HAL builds on top of platform specific LED device driver.
 
file  hal_log.h
 ACE Log Hal defines the required API implmentations from the platform to enable the ACE logging capability. Definition of the terms: *log buffer: a repository where logs with common log buffer ID are collected. *log message: one text or binary log in the raw format without metadata. *log entry: a log with meta datas such as timestamp, uid, tid.
 
file  hal_metric.h
 ACE metric platform API.
 
file  hal_ota.h
 A list of interfaces that define the platform-specific functionality required by ACE OTA client in order to successfully synchronize (with the configured OTA service), download, and install software updates.
 
file  hal_power.h
 ACE Power HAL provides the interfaces to require low level code to configure HW for different power state needs. This layer will serve the commands/requests from the Power Service middleware.
 
file  hal_thermal.h
 ACE Thermal HAL provides the interfaces to require thermal zone, cooling zone data, pass thermal policy to low level for linux system receive thermal callback for thermal state change. This layer layer will serve as the commands/requests from the Thermal Service middleware.
 
file  hal_virtualkey.h
 ACE Virtual Key HAL is designed to accept the key events coming from the peripheral devices, such as Bluetooth headphone or remote control. When user presses a key on such device, a key event will be generated then processed by this HAL.
 
file  hal_wifi.h
 HAL WiFi API to be implemented per platform to support WiFi functionality.
 
file  hal_wifi_ffs.h
 HAL WiFi API implemented per platform to support WiFi FFS functionality.
