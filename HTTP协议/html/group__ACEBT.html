Modules

 ACE Bluetooth APIs
 ACE Bluetooth APIs provides the interfaces to use Bluetooth functionality on the current platform. The API supports multiple clients in allowing them to concurrently access Bluetooth APIs.
 
 Bluetooth callbacks
 ACE Bluetooth callbacks define the callback events for various Bluetooth modules. Note that the thread context of the Bluetooth callback will be in the middleware context and hence will be blocking the middleware call. It is recommended by all clients to do a context switch before proceeding in the callback context.
 
 Data Structures and Enums for Bluetooth
 Data structure and enum definitions for Bluetooth clients to use in order to invoke APIs and handle callbacks.
 

Detailed Description

ACE Bluetooth Middleware.

ACE Bluetooth middleware provides a set of APIs to use Bluetooth (Classic/BLE) on the current platform. The ACE Bluetooth APIs are session-based APIs. Application clients will need to open a DUAL_MODE_SESSION or a BLE_ONLY_SESSION before calling any of the ACE Bluetooth APIs.
