ace_status_t aceCli_processCmd ( char *  msg)

CLI entry function to process a command.

Options:
?: Prints out help of all module_cmd_t in CLI_MODULE_CMDS
<command> ? : Prints out help of all sub_cmds for command in module_cmt_t->sub_cmds
<command> invalidSubCommand : Prints out help of <command>
<command> <subcommand leaf>=""> ? : Prints out help of sub_command
<command> <subcommand leaf>=""> <anything>: Responsibility of func to handle

Parameters
[in]msgnull deliminated string, note this message will be modified
Note
Parse msg by dilineating by line (each
is one message to parse, if no
, then just one line) then separating by space.
Returns
ACE_STATUS_OK if arguments were good, else CLI status from ace_status_t

Pretend locked is the same as not found to not hint to unauthorized users.

If its not 0, then get rid of trailing space.

Move ptr to end of message in preparation to search for next command.

Do something with the returned function. If it's a leaf call its function.

If the first arg is '?' then print out help instead of calling function.

◆ aceCli_setIsLocked()

void aceCli_setIsLocked ( aceCli_isLockedFunc_t *  is_locked_func)

Sets the islocked function.

If set, calls is_locked_func() and only calls restricted (ACE_CLI_RESTRICTED = 1) commands if this is false. If not set, all cli functions will always be called. This allows for a sudo-like experience of commands labeled restricted, and it is up to the implementer to determine what is_locked looks like.

Parameters
[in]is_locked_funcFunction to call to check if device is locked. Can be NULL.
Returns
None
