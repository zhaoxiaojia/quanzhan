aceBT_status_t acebt_pair ( aceBT_bdAddr_t *  p_remoteDevice,
aceBT_transportType_t  transport 
)

API to initiate a bond to the remote Bluetooth device on a given transport. Once bonding is done,the Bluetooth middleware may proceed to do the profile connections as well if the bonding is done over classic (BR/EDR) transport. The bond state and the connection state events are broadcasted to all clients registered in aceBT_openSession() call.

Parameters
[in]p_remoteDeviceMAC Address of remote device name
[in]transportTransport type for pairing
Returns
ACEBT_STATUS_SUCCESS if success; otherwise, appropriate bt_status_t error.

Once bonding is done,the Bluetooth middleware may proceed to do the profile connections as well if the bonding is done over classic (BR/EDR) transport. The bond state and the connection state events are broadcasted to all clients registered in aceBT_openSession() call.

Parameters
[in]p_remoteDeviceMAC Address of remote device name
[in]transportTransport type for pairing
Returns
ACEBT_STATUS_SUCCESS if success
ACEBT_STATUS_NOMEM if ran out of memory
ACEBT_STATUS_BUSY if profile is busy connecting another device
ACEBT_STATUS_DONE if remote device is already bonded
ACEBT_STATUS_PARM_INVALID if request contains invalid parameters
ACEBT_STATUS_NOT_READY if server is not ready
ACEBT_STATUS_FAIL for all other errors
