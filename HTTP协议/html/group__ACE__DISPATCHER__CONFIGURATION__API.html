const char *  name,
int32_t  queue_size,
int32_t  stack_size,
int32_t  thread_priority 
)

Create an instance of the dispatcher.

This function creates an instance of the dispatcher by creating a thread and a queue. It returns a dispatcher handle back to the caller that can be used for subsequent operations.

Note
The queue_size and thread priority must be appropriately chosen based on the modules that are expected to run inside the dispatcher.
Warning
Using copies of the dispatcher handle results in undefined behavior.
Parameters
[in]dpPointer to the memory for the dispatcher handle
[in]nameName of the queue
[in]queue_sizeNumber of elements in the queue
[in]stack_sizeStack size of the thread
[in]thread_priorityPriority of the thread
Returns
ACE_STATUS_OK on success, else error code from ace_status_t.

Print debugging information about dispatcher.

This function prints a list of dispatchers and the modules registered to each one. The debug information is for all dispatchers in the current process. RTOS is treated as though there is only one process in the entire system.

Returns
The sum of dispatchers and modules found or a status from ace_status_t on error

◆ aceDispatcher_deregisterModule()

ace_status_t aceDispatcher_deregisterModule ( aceDispatcher_dispatchHandle_t *  h)

Deregister a module from a dispatcher.

This function deregisters a module from an instance of the dispatcher. Once deregistered, the module's on_dereg cb is called if not NULL.

Note
For any message already in the dispatch queue would still result in its on_msg cb getting called. However, modules may track the fact that on_dereg cb has been called and drop the message.
Parameters
[in]hModule dispatch handle
Returns
ACE_STATUS_OK on success, else error code from ace_status_t.

◆ aceDispatcher_destroy()

ace_status_t aceDispatcher_destroy ( aceDispatcher_config_t *  dp)

Destroy an instance of the dispatcher.

This function destroys an instance of the dispatcher by ending the thread and deleting the queue. All registered modules must have been unregistered previously.

Note
This function blocks and may only be called from outside the dispatcher itself. Do not post a message to the dispatcher that in turn calls destroy. Instead directly call this api from an external user thread.
Parameters
[in]dpPointer to the dispatcher handle
Returns
ACE_STATUS_OK on success, else error code from ace_status_t.

ace_status_t aceDispatcher_join ( aceDispatcher_config_t *  dp)

Join the dispatcher thread.

This function joins the underlying dispatcher thread with the calling thread. It blocks the calling thread until the dispatcher thread is done. Hence,this call blocks indefinitely as a dispatcher thread never exits.

Parameters
[in]dpDispatcher handle
Returns
ACE_STATUS_OK on success, else error code from ace_status_t.

Register a module with a dispatcher.

This function registers a module with an instance of the dispatcher. Once registered, the module's on_reg callback is called with a reference to the dispatch handle. Modules can then schedule work for execution in the dispatcher after this point if registration is successful.

Parameters
[in]dpPointer to the dispatcher handle
[in]modPointer to the dispatch handler interface struct
[in,out]hModule dispatch handle
Returns
ACE_STATUS_OK on success, else error code from ace_status_t.
